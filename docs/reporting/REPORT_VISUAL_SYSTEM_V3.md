## Why This File Was Demoted

This file implicitly treated:

- one earlier six-page flow
- one previous v3 page mapping
- one tighter visual structure

as if that were the active visual truth for all reporting. That is no longer the current model.

The new governance layer now distinguishes between:

- canon that defines what the report is
- visual governance that defines how it looks
- active blueprint assets that support implementation later

## What Still Remains Useful

These ideas still remain useful when they do not conflict with the active canon:

- disciplined chart use
- appendix separation
- strong page hierarchy
- premium but compact management readability
- caution against placeholder copy and mixed product semantics

## What Is No Longer Active In This File

The following are no longer active claims from this file:

- any exact six-page main report assumption as default truth
- any suggestion that this file can set report architecture
- any mapping that would override the current ExitScan recipe or compress other products into that same recipe

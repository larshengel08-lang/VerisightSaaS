# Implementation Operator Checklist

Gebruik deze checklist voor de assisted implementation route vanaf klantakkoord tot eerste managementwaarde.

## 1. Route en setup

- [ ] Productroute bevestigd: ExitScan of RetentieScan
- [ ] Delivery mode bevestigd: baseline of live
- [ ] Organisatie aangemaakt of gecontroleerd
- [ ] Campaignnaam, scan_type en add-ons gecontroleerd
- [ ] Contactpersoon en eerste managementdoel expliciet vastgelegd

## 2. Klantbestand en import

- [ ] Klantbestand ontvangen met minimaal `email`
- [ ] Preview / dry run uitgevoerd
- [ ] Dubbelen, foutregels en ontbrekende e-mails opgelost
- [ ] Bij segment deep dive: `department` en `role_level` gecontroleerd
- [ ] Bij RetentieScan: metadata voldoende voor segmentatie en latere validatie
- [ ] Besluit genomen: direct uitnodigen of eerst klaarzetten

## 3. Invite en activatie

- [ ] Respondentinvites verstuurd of bewust uitgesteld
- [ ] Mailtemplate gecontroleerd op naam, organisatie en link
- [ ] Eventuele resend of reminderlogica afgestemd
- [ ] Klantdashboardtoegang uitgenodigd
- [ ] Activatie daadwerkelijk bevestigd

## 4. Launch en acceptatie

- [ ] Testsurvey handmatig doorlopen
- [ ] Bedankpagina gecontroleerd
- [ ] Minimaal 5 responses voor eerste veilige detailweergave
- [ ] Geen incomplete scoredata in opgeslagen responses
- [ ] Dashboardstats werken
- [ ] PDF-rapport werkt
- [ ] Eerste dashboardread of rapportuitleg ingepland
- [ ] Eerste managementgebruik bevestigd

## 5. Extra aandacht bij live routes

- [ ] Live-route alleen gebruikt als bewuste vervolgroute
- [ ] Timing en eigenaarschap nogmaals bevestigd
- [ ] Uitnodigingen niet automatisch verstuurd zonder laatste controle
- [ ] Eerste contactmoment na livegang expliciet ingepland

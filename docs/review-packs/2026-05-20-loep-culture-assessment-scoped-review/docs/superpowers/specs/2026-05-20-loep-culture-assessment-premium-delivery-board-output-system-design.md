# Loep Culture Assessment Premium Delivery & Board Output System

**Date:** 2026-05-20  
**Status:** first-wave design spec  
**Primary scope:** `Loep Culture Assessment / Loep Cultuurbeeld`  
**Canonical route id:** `culture_assessment`

---

## 1. Purpose

This spec defines the **Premium Delivery & Board Output System** for `Loep Culture Assessment / Loep Cultuurbeeld`.

Its purpose is to turn the route from:

- a strong baseline product with some real and some blueprint outputs

into:

- a coherent premium output family that is commercially demoable
- operationally deliverable in guided board and HR contexts
- governance-safe across executive, HR, and cascade layers

This spec must answer:

- which outputs belong to the standard product family
- which outputs are standard versus governed or enterprise-only
- what quality bar each output must meet
- how all outputs stay structurally and linguistically aligned
- what each output may and may not claim
- what it means for an output to be `blueprint_ready`, `demo_asset_ready`, or `commercial_delivery_ready`

This spec does not create a design system for the full Loep suite.
It defines the premium delivery layer for one route only.

---

## 2. Strategic Role

Inside the `Loep Culture Assessment` operating system, Premium Delivery & Board Output Excellence is the main commercial differentiator after Methodology Authority.

Its role is to ensure that the product feels:

- more board-usable than a classic MTO
- more guided and premium than generic survey software
- more disciplined than dashboard-heavy employee listening tooling

Without this layer, the route risks becoming:

- methodologically thoughtful but commercially underwhelming
- operationally real but visually and narratively inconsistent
- believable in product logic but not persuasive enough in executive buying contexts

This stream therefore exists to make premium delivery:

- repeatable
- governed
- category-serious
- visibly higher quality than commodity survey reporting

---

## 3. Core Delivery Principle

The output system follows one primary rule:

> Every output must support the same executive reading spine, even when the format, audience, and depth differ.

That means the product may use different artifacts for:

- board
- HR
- manager cascade
- sales demo

but those artifacts may not drift into different product stories.

All outputs must remain compatible with:

- the `Loep Cultuurbeeld Methode`
- the `Loep Culture Index` as navigation signal
- fixed domain logic
- governed release rules
- non-causal, non-ranking, non-predictive interpretation boundaries

## 5. Standard Versus Governed Outputs

The output family must remain explicit about standard inclusion versus governed expansion.

### 5.1 Standard baseline outputs

Included in the default baseline package:

- board report PDF
- executive one-pager
- board-read facilitator script
- guided board-read session

### 5.2 Governed outputs

Included only where delivery package and governance allow:

- HR appendix PDF
- segment summary export
- manager cascade handout

### 5.3 Demo and sales support outputs

Used for explanation, proposal support, or internal preparation:

- boardroom deck
- sample output pack
- demo organization
- sample invite and what-you-receive assets

## 6. Canonical Executive Reading Spine

All premium outputs must map back to the same canonical executive reading sequence.

The canonical order remains:

1. Responsbasis & meetdekking
2. Executive culture read
3. Loep Culture Index
4. Board attention points
5. Domeinbeeld
6. Patronen in samenhang
7. Segmentcontrasten
8. Verdiepingslagen
9. Open signalen
10. Board-read & vervolgritme
11. Rapport, export & methodiek

### 6.1 Condensed versus full sequence

Not every artifact must render all eleven blocks at equal depth.

Allowed variants:

- `full sequence`: dashboard and boardroom deck
- `condensed executive sequence`: board report PDF and executive one-pager
- `governed deepening sequence`: HR appendix
- `cascade-safe sequence`: manager handout

But every artifact must still point back to the same logic:

- response basis before interpretation
- index as navigation signal, not verdict
- domains before action
- governance boundaries before local or cascading use

---

## 7. Readiness States

Every output must use one explicit readiness state.

### 7.1 `blueprint_ready`

Meaning:

- structure is defined
- copy boundaries are defined
- audience and governance rules are defined
- not yet a final commercial artifact

### 7.2 `demo_asset_ready`

Meaning:

- usable in guided sales or review
- visually and narratively coherent enough to show externally
- still may require human facilitation or manual tailoring

### 7.3 `commercial_delivery_ready`

Meaning:

- can be used in a real client delivery context
- copy, structure, governance, and audience fit are stable enough for repeatable use
- no critical ambiguity remains about claims, thresholds, or handoff rules

## 8. Artifact Specifications

## 8.1 Board report PDF

Purpose:

- provide a formal executive read after the baseline closes and releases

Required structure:

- executive framing and scope
- response basis and coverage
- Loep Culture Index as navigation signal
- condensed domain picture
- board attention points
- safe segment contrast summary where allowed
- what not to conclude
- board-read and follow-on route framing
- method and governance note

Quality bar:

- readable by a board sponsor without live dashboard dependence
- clearly more serious than a classic survey PDF
- does not require the buyer to become an analyst

### 8.1.1 Claim and governance rules

The board report may:

- summarize broad organization-level patterns
- show safe segment contrast where governance allows
- explain what requires further board or HR attention

The board report may not:

- claim culture is good or bad in a total sense
- assign blame to managers or teams
- make causal claims
- show unsafe local segment layers

## 8.2 Boardroom PowerPoint deck

Purpose:

- support guided board-read and executive discussion

Required structure:

1. title and context
2. what was measured
3. response and coverage
4. executive read
5. Loep Culture Index as navigation signal
6. domain picture
7. 3-5 board attention points
8. segment contrasts where safe
9. what not to conclude
10. board decision questions
11. follow-on route options

Quality bar:

- usable in an executive meeting without extra explanation
- every slide reinforces the same method and governance posture
- no slide reads like a ranking or health-score deck

## 8.3 Executive one-pager

Purpose:

- deliver a compact summary that can stand alone

Required structure:

- what this product measured
- what the executive receives
- the index reading rule
- 3-5 main visible patterns
- what this is not
- follow-on framing

Quality bar:

- understandable without facilitator narration
- short enough for executive reuse
- precise enough not to create drift from the core route

## 8.4 HR appendix PDF

Purpose:

- provide a governed deeper layer for HR after the executive read

Required structure:

1. scope and governance boundary
2. response basis per safe segment layer
3. domain picture per safe segment
4. segment contrasts in samenhang
5. safe open-signal clusters when enabled
6. suppressed segments and why they stay hidden
7. HR follow-up boundaries

Quality bar:

- gives enough depth without exposing unsafe detail
- helps HR interpret governed contrast, not hunt for local blame
- explicitly documents hidden and suppressed layers

## 8.5 Manager cascade handout

Purpose:

- support safe communication after board and HR interpretation

Required structure:

- what was measured at organization level
- what can safely be discussed locally
- what the manager should not conclude
- how to structure a follow-up conversation
- where escalation or HR support is needed

Quality bar:

- can be used without creating ranking behavior
- helps conversation without forcing score defense
- never behaves like a manager performance artifact

### 8.5.1 Governance rules

The manager cascade handout may:

- explain what was measured at organization level
- support safe team communication
- structure follow-up conversation without score defense

The manager cascade handout may not:

- compare teams as a ranking
- ask managers to defend low scores
- discuss individual signals
- show local scores below threshold

## 8.6 Board-read facilitator script

Purpose:

- structure a 60-90 minute guided board-read

Required structure:

- session goal
- participants
- preparation
- opening rules
- walk-through sequence
- decision questions
- follow-up boundary and handoff

Quality bar:

- supports a 60-90 minute board-read without improvising the product logic
- avoids causal diagnosis and manager blame
- keeps governance and interpretation boundaries visible throughout

## 8.7 Sample output pack

Purpose:

- show the product without requiring a live environment

Required contents:

- sample board report PDF
- sample boardroom deck
- sample executive one-pager
- sample HR appendix
- sample dashboard screenshots or demo environment
- sample invite mail
- sample board-read agenda
- sample "what you receive after completion" page

Quality bar:

- enough to explain the route to a board or HR buyer in ten minutes
- commercially coherent
- explicitly marked where an asset is blueprint-only or guided-demo-only

## 9. Claim-Safe Output Rules

All artifacts must share the same claim-safe core.

Allowed statements:

- broad culture and engagement baseline
- fixed domain model
- index as navigation signal
- governance-bounded result reading
- safe contrast where thresholds allow

Forbidden or tightly bounded statements:

- culture is good or bad
- manager X is weak
- this proves causality
- this predicts individual behavior
- this benchmark proves relative superiority in v1

The same claim rules must appear consistently across:

- board outputs
- HR outputs
- manager/cascade outputs
- sales/demo outputs

## 10. Governance-Safe Audience Model

The output family uses four bounded audience layers.

### 10.1 Executive layer

Includes:

- board report
- board deck
- executive one-pager
- board-read session

Depth:

- high-level, structured, decision-oriented

### 10.2 HR governed layer

Includes:

- HR appendix
- governed segment summary export

Depth:

- deeper, still thresholded and governed

### 10.3 Manager cascade layer

Includes:

- manager cascade handout only where allowed

Depth:

- bounded, communication-oriented

### 10.4 Sales and demo layer

Includes:

- sample output pack
- guided demo artifacts

Depth:

- commercially legible, never overclaiming delivery maturity

## 11. Enterprise and MKB Deployment Profiles

The same output logic applies across deployment profiles, but depth differs.

### 11.1 Enterprise profile

Standard:

- board report
- executive one-pager
- guided board-read

Governed expansion:

- HR appendix
- segment summary export
- deeper board deck and governed drilldown explanation

### 11.2 MKB profile

Standard:

- same core instrument
- same board report logic
- same executive one-pager
- same guided board or MT-read

Boundaries:

- fewer segment layers by default
- no default named manager layer
- no "lite" positioning
- MT-read is more important than dashboard depth

## 12. Ownership and Approval

Every output must have:

- a content owner
- a governance approver
- a delivery owner
- a readiness state owner

At minimum, the output family must define:

- who may approve `commercial_delivery_ready`
- who may downgrade an output to `demo_asset_ready`
- who owns the claim-safe copy source of truth

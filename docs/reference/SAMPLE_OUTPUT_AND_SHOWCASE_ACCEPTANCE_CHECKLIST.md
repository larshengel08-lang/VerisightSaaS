# Sample Output And Showcase Acceptance Checklist

Gebruik deze checklist wanneer actieve voorbeeldrapporten, preview-copy, showcase-routing of buyer-facing sample-assets wijzigen.
Laatste update: 2026-04-15

## Buyer-facing samples

- [ ] ExitScan-voorbeeldrapport opent zonder vertrouwelijkheidsframing.
- [ ] RetentieScan-voorbeeldrapport opent zonder vertrouwelijkheidsframing.
- [ ] Beide actieve pdf's gebruiken expliciet fictieve data.
- [ ] Beide actieve pdf's volgen dezelfde managementstructuur als live output.
- [ ] ExitScan leest als primaire showcase voor vertrekduiding.
- [ ] RetentieScan leest als complementaire verification-first showcase voor behoud op groepsniveau.
- [ ] Geen actief sample-asset verkoopt diagnose, predictor, performance-instrument of bewezen causaliteit.

## Website showcase

- [ ] Home gebruikt de portfolio-preview alleen als routekiezer en niet als vervanging van het voorbeeldrapport.
- [ ] Producten-overzicht blijft teaser-first.
- [ ] ExitScan-productdetail biedt een duidelijke link naar het actieve ExitScan-voorbeeldrapport.
- [ ] RetentieScan-productdetail biedt een duidelijke link naar het actieve RetentieScan-voorbeeldrapport.
- [ ] Tarieven gebruikt sample-output alleen als deliverable-proof.
- [ ] Vertrouwen gebruikt sample-output alleen als trustproof.

## Parity

- [ ] Preview-copy gebruikt dezelfde managementtaal als de actieve sample-pdf's.
- [ ] Factorcards en dashboardrijen in de preview suggereren geen rijkere output dan de actieve pdf's.
- [ ] Publieke pdf-mirrors in `frontend/public/examples` zijn gelijk aan de actieve docs-versies.
- [ ] Legacy `*_35_fictief.pdf` wordt nergens meer als leidende showcase gebruikt.

## Sales and trust

- [ ] Sales-assets gebruiken sample-output als eerste prooflaag voor deliverable-kwaliteit.
- [ ] Sample-output wordt niet verkocht als case-proof of effectbewijs.
- [ ] Trustframing blijft zichtbaar: gegroepeerde managementduiding, privacygrenzen en bewijsstatus.

## Refresh governance

- [ ] Bij wijziging in report contracts is de actieve sample-output opnieuw gegenereerd.
- [ ] Bij wijziging in preview-copy is de showcase-parity opnieuw beoordeeld.
- [ ] Bij wijziging in routing of buyer-facing copy zijn pricing- en trustingangen mee gecontroleerd.
- [ ] `docs/examples/README.md` en `docs/reference/SAMPLE_OUTPUT_AND_SHOWCASE_SYSTEM.md` zijn nog actueel.
- [ ] `PROMPT_CHECKLIST.xlsx` weerspiegelt de werkelijke repo-status van dit traject.

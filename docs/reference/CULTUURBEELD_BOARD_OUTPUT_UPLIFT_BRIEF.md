## 2. Mercer-to-Loep translation matrix

| Mercer-style element | Decision | Loep equivalent / rule |
| --- | --- | --- |
| Report roadmap | `adopt` | Loep should use a clear board-read roadmap so the reader understands the sequence from response basis to signal, pattern, governance, and bounded follow-on. Use a calm page spine, not a consultant agenda wall. |
| Interpretation guidelines | `adopt` | Loep should make reading rules explicit: what the signal means, what it does not prove, how to read confidence, and why governed layers stay hidden. |
| Methodology scale page | `adapt` | Keep a concise method page, but do not mimic dense survey-method appendix behavior. Use a short “what was measured / how to read this” page tied to the fixed Loep method. |
| Dashboard summary | `adapt` | Loep should not copy a dashboard export page. Translate this into an editorial executive culture read and response/governance overview. |
| Dimension summary | `adopt` | Loep needs a strong domain picture page. Present domains in **bestuurlijke leesvolgorde**, not as a top-bottom ranking. |
| Action priority grid | `adapt` | Translate to **board attention points** and a bounded decision layer. Use “what deserves board attention now” rather than intervention matrix theater. |
| Key drivers | `adapt` | Do not use causal “driver” claims. Translate into **pattern logic**, recurring theme pairs, and priority hypotheses that remain descriptive. |
| Heat map | `adapt` | Use only as a governed, bounded contrast device and never as the default board page. Avoid dense corporate survey heat-map walls. A compact contrast lattice can appear later in HR appendix or governed release. |
| Item detail | `later` | Not default board output. Item-level pages are too dense for the board artifact and risk over-reading. Could appear later in an HR-governed appendix if explicitly released. |
| Comment summary | `adopt` | Loep should include a governed, role-safe text summary layer. No raw quote wall, no browsing behavior, no unsafe free text dump. |
| Union index | `reject` | Not part of Loep Culture Assessment. Loep is not a labor-relations index, not a benchmark product, and not a union climate tracker. |

### Translation principle

The Mercer-style deck shows that enterprise output maturity comes from:

- stronger narrative sequencing
- clearer explanation layers
- more explicit interpretation guidance
- richer but disciplined visual depth

Loep should copy **the maturity standard**, not the survey logic.

## 5. Output family architecture

| Artifact | Role | Audience | Standard or governed | Visual intensity | Data depth | Must not contain |
| --- | --- | --- | --- | --- | --- | --- |
| Board Report PDF | Self-readable board artifact and download-ready executive baseline report | Board, directie, CHRO | `standard` baseline artifact | High | Medium | Dense appendix logic, raw quote dumps, ranking tables, benchmark framing, manager-level exposure |
| Boardroom Deck | Guided board-read companion for facilitated discussion | Board, directie, facilitator | `standard` baseline artifact | Very high | Medium | Small-text report pages, dashboard-export screenshots, exploratory analytics feel |
| HR Appendix | Governed deepening layer behind the board artifact | CHRO, HR lead, governed internal readers | `governed` | Medium | High | Named manager exposure, unrestricted segment lattice, raw unsafe text, buyer-facing packaging as standard output |
| Manager Cascade Handout | Bounded cascade-safe communication artifact | Senior leaders, selected managers under governance | `governed` | Medium | Low to medium | Ranking of teams/managers, hidden segment data, raw item detail, blame language |
| Executive One-Pager | Condensed follow-up summary for later reuse | CEO, CHRO, sponsor | `later` standard derivative | High | Low | Full deep-dive logic, dense charts, appendix content, new interpretation claims |

### Family rules

- The **Board Report PDF** is the default executive artifact.
- The **Boardroom Deck** is the strongest visual artifact.
- The **HR Appendix** is not a default give-away; it stays governed.
- The **Manager Cascade Handout** must stay cascade-safe and non-ranking.
- The **Executive One-Pager** should be derived later from the same visual system, not designed as a separate style world.

## 6. Board Report Redesign Blueprint v1

## 4.1 Page 1 - Cover

- **Purpose:** establish premium tone and annual baseline seriousness
- **Data inputs:** organization name, campaign title, reporting period, route subtitle
- **Visual module:** editorial cover composition with strong title field and controlled atmosphere layer
- **Copy role:** position this as a bounded annual board baseline, not a survey export
- **Governance cue:** disciplined tone from the first page; no overt governance block needed here
- **Forbidden elements:** metrics, gauges, benchmark language, metadata tables, dashboard cards

## 4.2 Page 2 - What was measured / response basis

- **Purpose:** explain the fixed method and show the evidence base early
- **Data inputs:** ten fixed domains, invited count, completed count, response basis, visibility threshold note
- **Visual module:** calm method overview + response strip + compact governance frame
- **Copy role:** explain what the route measures and what the response base supports
- **Governance cue:** note that deeper detail opens only after safe release conditions
- **Forbidden elements:** custom-module feel, benchmark scale, dense methodology prose

## 4.3 Page 3 - Executive culture read

- **Purpose:** give the board a first integrated reading line before showing charts
- **Data inputs:** executive culture read summary, management question, bounded interpretation line
- **Visual module:** one dominant editorial narrative block
- **Copy role:** answer “what matters most in this baseline?”
- **Governance cue:** descriptive only; no causal proof and no intervention promises
- **Forbidden elements:** action checklist, diagnostic overclaim, consultant filler paragraphs

## 4.4 Page 4 - Loep Culture Index hero

- **Purpose:** position the Culture Index as navigation signal, not verdict
- **Data inputs:** Culture Index value, short interpretation line, domain context cue
- **Visual module:** hero signal field or editorial index band
- **Copy role:** explain what the index helps orient and what it does not prove
- **Governance cue:** no benchmark, no pass/fail, no health widget semantics
- **Forbidden elements:** gauge, speedometer, traffic-light widget, benchmark-position graphic

## 4.5 Page 5 - Board attention points

- **Purpose:** surface the 3-5 bounded attention lines that deserve board discussion
- **Data inputs:** board attention points, confidence chips, verify-next cues
- **Visual module:** strong attention-card cluster
- **Copy role:** translate the baseline into board-discussable issues without turning into an action tracker
- **Governance cue:** confidence means signal stability, not truth certainty or causal certainty
- **Forbidden elements:** intervention backlog, blame language, certainty theater

## 4.6 Page 6 - Domain picture / bestuurlijke leesvolgorde

- **Purpose:** show the broad culture profile across the ten domains
- **Data inputs:** domain scores, domain labels, short interpretation lines
- **Visual module:** editorial domain profile with subdued labels
- **Copy role:** guide the reader through the reading sequence
- **Governance cue:** order is reading sequence only, never ranking
- **Forbidden elements:** red/amber/green coloring, top/bottom framing, dominant rank numbers

## 4.7 Page 7 - Patterns in relation

- **Purpose:** elevate the read beyond single dimensions and into system patterns
- **Data inputs:** recurring theme pairs, related domain cues, bounded interpretive text
- **Visual module:** relationship diagram or paired narrative modules
- **Copy role:** explain what tends to appear together and why that matters descriptively
- **Governance cue:** descriptive pattern logic only; no causal arrows or driver proof
- **Forbidden elements:** correlation matrix feel, technical model diagram, “root cause” language

## 4.8 Page 8 - Segment contrasts if governed and released

- **Purpose:** show bounded differences only when safe and manually released
- **Data inputs:** released segment contrasts, approved labels, hidden/suppressed states
- **Visual module:** compact contrast cards or contrast lattice with governance-state rendering
- **Copy role:** state what can be said about released contrasts and what stays hidden
- **Governance cue:** unreleased or suppressed layers must appear as governance state, not silent absence
- **Forbidden elements:** league tables, winners/losers, best/worst segments, named manager visibility

## 4.9 Page 9 - What this does not prove

- **Purpose:** actively prevent overclaim and false certainty
- **Data inputs:** fixed boundary statements, interpretation limits
- **Visual module:** designed boundary statement cluster
- **Copy role:** explain what cannot be concluded from this baseline
- **Governance cue:** no causal proof, no individual prediction, no manager ranking, no benchmark claim
- **Forbidden elements:** legal boilerplate dump, defensive tone, contradictory wording

## 4.10 Page 10 - Board-read and follow-on direction

- **Purpose:** close the report with disciplined discussion and bounded next-route choice
- **Data inputs:** board-read questions, follow-on options, Pulse eligibility note, deeper governed work option
- **Visual module:** decision-question block + route decision matrix
- **Copy role:** turn results into bounded board decisions, not open-ended task theater
- **Governance cue:** no route opens automatically; Pulse is conditional and not a patch for unclear baseline reading
- **Forbidden elements:** workflow platform feel, action-center mimicry, automatic continuation logic

## 7. Boardroom Deck Redesign Blueprint v1

## 5.1 Slide 1 - Opening / purpose

- **Presenter purpose:** open the board-read with seriousness and scope
- **Audience question answered:** why are we here and what kind of artifact is this?
- **Visual center:** full-slide editorial opening composition
- **Supporting copy:** annual baseline framing and one purpose statement
- **Speaker note:** frame the session as guided and bounded
- **Governance boundary:** no metrics before scope is clear
- **Transition to next slide:** move to what the route actually measured

## 5.2 Slide 2 - What was measured and what was not

- **Presenter purpose:** distinguish fixed Loep method from broader survey or listening products
- **Audience question answered:** what exactly is in scope here?
- **Visual center:** fixed-method system view with ten domains
- **Supporting copy:** fixed domains, annual cadence, non-benchmark-first, no custom build-your-own survey logic
- **Speaker note:** build confidence through clarity
- **Governance boundary:** no new measurement logic implied
- **Transition to next slide:** move from method scope to evidence quality and governance

## 5.3 Slide 3 - Response and governance

- **Presenter purpose:** establish the evidence basis and visibility rules
- **Audience question answered:** what is the data foundation and what remains hidden?
- **Visual center:** response strip + governance boundary block
- **Supporting copy:** invited, completed, response basis, hidden-layer note
- **Speaker note:** explain that governance is part of quality, not a limitation pasted on top
- **Governance boundary:** `Wat getoond mag worden`, `Wat verborgen blijft`, `Waarom dat zo is`, `Wat je hier niet uit mag concluderen`
- **Transition to next slide:** move from evidence basis to the executive read

## 5.4 Slide 4 - Executive read

- **Presenter purpose:** give the room the first integrated reading line
- **Audience question answered:** what is the main executive reading of this baseline?
- **Visual center:** statement-led narrative block
- **Supporting copy:** concise executive read + one management question
- **Speaker note:** set the frame for the rest of the deck
- **Governance boundary:** descriptive only, not causal
- **Transition to next slide:** move to the Culture Index hero

## 5.5 Slide 5 - Culture Index as navigation signal

- **Presenter purpose:** explain the central signal without overclaim
- **Audience question answered:** what is the broad signal?
- **Visual center:** editorial index band / vertical signal field / abstract signal field
- **Supporting copy:** index value, interpretation line, what the index is not
- **Speaker note:** say explicitly that this is a navigation signal, not a final verdict
- **Governance boundary:** no gauge, no pass/fail, no benchmark position
- **Transition to next slide:** move to the domain picture that gives the index shape

## 5.6 Slide 6 - Domain picture

- **Presenter purpose:** show the broad organization-wide domain profile
- **Audience question answered:** how does the culture profile look across the fixed domains?
- **Visual center:** editorial domain profile
- **Supporting copy:** short reading-sequence line and minimal interpretation cues
- **Speaker note:** walk the room through the sequence, not a ranking
- **Governance boundary:** no top/bottom framing
- **Transition to next slide:** move to attention point 1

## 5.7 Slide 7 - Board attention point 1

- **Presenter purpose:** isolate the first board issue so it lands clearly
- **Audience question answered:** what deserves board attention first?
- **Visual center:** one strong attention card
- **Supporting copy:** priority reason and verify-next cue
- **Speaker note:** keep the read bounded and evidence-linked
- **Governance boundary:** confidence is stability, not certainty
- **Transition to next slide:** move to attention point 2

## 5.8 Slide 8 - Board attention point 2

- **Presenter purpose:** give the second attention line equal narrative weight
- **Audience question answered:** what is the second bounded issue to discuss?
- **Visual center:** one strong attention card
- **Supporting copy:** priority reason and verify-next cue
- **Speaker note:** connect this to the domain/pattern logic already shown
- **Governance boundary:** no intervention theater
- **Transition to next slide:** move to attention point 3

## 5.9 Slide 9 - Board attention point 3

- **Presenter purpose:** complete the core attention trilogy
- **Audience question answered:** what is the third bounded issue to discuss?
- **Visual center:** one strong attention card or shaped editorial block
- **Supporting copy:** priority reason and verify-next cue
- **Speaker note:** prepare the room for any released contrast layer
- **Governance boundary:** no hidden-layer inference
- **Transition to next slide:** move to segment contrast if released

## 5.10 Slide 10 - Segment contrast if released

- **Presenter purpose:** show governed contrast only where manually released
- **Audience question answered:** are there safe released differences worth noting?
- **Visual center:** compact contrast module or governance-state panel
- **Supporting copy:** one short interpretation of visible contrasts or one short note on why they remain hidden
- **Speaker note:** keep this bounded and low-drama
- **Governance boundary:** only approved released segments, no league table, no winners/losers
- **Transition to next slide:** move to what must not be concluded

## 5.11 Slide 11 - What not to conclude

- **Presenter purpose:** actively prevent misuse and over-reading
- **Audience question answered:** what conclusions would be methodologically wrong?
- **Visual center:** boundary statement cluster
- **Supporting copy:** no causality, no manager ranking, no individual prediction, no benchmark claim
- **Speaker note:** present this as quality discipline, not defensiveness
- **Governance boundary:** this slide is itself a governance boundary
- **Transition to next slide:** move to the real board decision questions

## 5.12 Slide 12 - Decision questions

- **Presenter purpose:** turn the deck into a guided board discussion
- **Audience question answered:** what should the board actually decide or clarify?
- **Visual center:** question matrix or editorial question blocks
- **Supporting copy:** 3-5 fixed board questions
- **Speaker note:** facilitate discussion, not an action tracker workflow
- **Governance boundary:** no automated intervention path
- **Transition to next slide:** move to bounded follow-on options

## 5.13 Slide 13 - Follow-on route options

- **Presenter purpose:** close with disciplined next-route choices
- **Audience question answered:** what can we do after this baseline?
- **Visual center:** route decision matrix
- **Supporting copy:** no immediate next route, deeper governed work, Pulse follow-on, another Loep route
- **Speaker note:** emphasize that nothing opens automatically
- **Governance boundary:** Pulse is conditional and not a patch for unclear baseline interpretation
- **Transition to next slide:** session closes into guided discussion or decision

## 8. Visual quality rules

## 6.1 What to learn from enterprise depth

Loep should learn from the reference deck that enterprise maturity comes from:

- strong report roadmap and wayfinding
- explicit interpretation guidance
- visible methodology explanation
- layered summary structure
- disciplined deepening hierarchy
- clear movement from signal to pattern to decision

## 6.2 What to avoid from dense corporate survey output

Loep should avoid:

- overpacked heat-map walls
- endless item-detail tables
- appendix logic invading the board artifact
- small-text consultant density
- “every metric deserves a box” layout behavior

## 6.3 How Loep should feel

Loep should feel:

- premium
- editorial
- warm
- boardroom-readable
- calmer and more human than standard survey decks
- more boutique and less templated

## 6.4 How to avoid dashboard-export aesthetics

- no raw product screenshots as report pages
- no widget grids
- no KPI tile fields as primary layout
- no export-looking charts with default library styling
- every page needs one visual center, not six equal modules

## 6.5 How to avoid ranking / traffic-light / benchmark overreach

- no red/amber/green score semantics
- no top/bottom domain storytelling
- no benchmark position visuals
- no “best/worst segment” language
- no driver claims that imply causality

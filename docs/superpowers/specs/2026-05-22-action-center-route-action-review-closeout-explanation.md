# Action Center Route -> Action -> Review -> Closeout Explanation

## Purpose

This explanation describes the canonical operating flow for Action Center inside Loep.

## 1. Route

A route opens from an approved scan context and becomes the bounded container for post-scan follow-through. The route holds the management question, operating context, and closeout path.

## 2. Action

When execution is needed, a manager or approved participant adds a bounded action inside that route. The action exists to move one concrete part of route follow-through forward without creating a separate work system.

## 3. Review

Each action is reviewed inside a defined rhythm. Review checks whether follow-through happened, stalled, or needs continuation. Review keeps execution interpretable for both managers and HR.

## 4. Closeout

HR decides whether the route is ready for closeout or should continue. Closeout happens when the route no longer requires active follow-through under its bounded purpose.

## Why This Flow Matters

- it keeps execution tied to the route instead of to free-form task sprawl
- it keeps managers involved without making them workflow owners
- it keeps HR in charge of continuation and closeout decisions
- it keeps follow-through canonical inside Action Center

## What This Flow Is Not

- not a project plan
- not a task tree
- not workflow automation
- not a case-management flow

## Plain-Language Summary

The route defines the follow-through container, actions move it forward, reviews test whether movement happened, and HR decides when the route is complete.

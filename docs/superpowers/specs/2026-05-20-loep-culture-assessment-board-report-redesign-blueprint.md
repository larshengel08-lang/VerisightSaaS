# Loep Culture Assessment Board Report Redesign Blueprint

**Date:** 2026-05-20  
**Status:** board-report redesign blueprint  
**Primary scope:** `Loep Culture Assessment / Loep Cultuurbeeld`  
**Canonical route id:** `culture_assessment`

---

## 1. Purpose

This blueprint translates the `Loep Culture Assessment Premium Output Visual Redesign` spec into a **page-by-page redesign structure** for the `board report PDF`.

Its purpose is to define, for each page:

- page title
- purpose
- data inputs
- layout composition
- visual modules
- copy blocks
- governance cues
- forbidden elements
- acceptance criteria

This blueprint is intentionally limited to the `board report PDF`.

It does not redesign:

- the application UI
- the HR appendix
- the manager cascade handout
- benchmark layers
- self-serve outputs

---

## 2. Report-Level Design Rules

These rules apply to every page in the report.

### 2.1 Format

- portrait A4
- PDF export first
- self-readable without facilitator dependence
- print- and download-worthy

### 2.2 Grid and margins

- 12-column editorial grid
- generous outer margins
- stable inner gutters
- no dashboard-style widget packing

### 2.3 Typography

- display tier for cover and signal hero
- heading tier for page titles and attention titles
- subheading tier for labels and governance modules
- body tier for narrative explanation
- caption tier for annotations and interpretation limits

### 2.4 Color and chart behavior

- warm, human, premium palette
- restrained chart palette roles
- no red/amber/green ranking logic
- no benchmark position logic
- suppressed/unreleased material shown as governance state, not as absence

### 2.5 Narrative behavior

Every page must answer a boardroom question.

Every page must have:

- one clear visual center
- one dominant reading line
- one explicit interpretation boundary where needed

---

## 3. V1 Page Sequence

The board report PDF v1 page sequence is fixed as:

1. Cover
2. Executive opener
3. Response basis & governance frame
4. Loep Culture Index hero
5. Board attention points
6. Domain profile
7. Pattern logic
8. Segment contrasts, if governed and released
9. Follow-on decision
10. Method & boundaries

---

## 4. Page Blueprint

## 4.1 Page 1 — Cover

### Page title

`Loep Culture Assessment / Loep Cultuurbeeld`

### Purpose

- establish premium first impression
- position the report as a boardroom artifact
- signal annual baseline seriousness before any data appears

### Data inputs

- campaign title
- organization name
- reporting period or release window
- route subtitle

### Layout composition

- full-page editorial composition
- one dominant title block
- one quiet contextual subline
- strong negative space
- optional abstract background field or subtle texture

### Visual modules

- cover title panel
- route context line
- restrained premium atmosphere layer

### Copy blocks

- product name
- annual baseline subtitle
- short board/directie/HR framing

### Governance cues

- none as a separate block
- tone should already imply disciplined, bounded output

### Forbidden elements

- metrics on the cover
- dashboard cards
- dense metadata table
- benchmark language
- traffic-light styling

### Acceptance criteria

- immediately feels like a premium board artifact
- does not feel like a software export
- title hierarchy is strong and calm

---

## 4.2 Page 2 — Executive Opener

### Page title

`Executive culture read`

### Purpose

- orient the reader in one page
- state what the baseline is
- frame what this report is meant to help a board discuss

### Data inputs

- executive culture read summary
- management question
- route description
- high-level release state

### Layout composition

- upper narrative block
- adjacent or lower governance frame
- compact response/context strip
- one strong opening statement

### Visual modules

- editorial narrative block
- response strip
- governance note block

### Copy blocks

- concise executive read
- management question
- what this route is / is not

### Governance cues

- note that this is a bounded annual baseline
- note that deeper layers remain governed

### Forbidden elements

- detailed charts
- ranked domain tables
- segment comparison table
- intervention advice

### Acceptance criteria

- reader understands the purpose of the report within one page
- page feels editorial, not administrative
- governance frame feels designed, not bolted on

---

## 4.3 Page 3 — Response Basis & Governance Frame

### Page title

`Responsbasis en governancekader`

### Purpose

- show what data foundation exists
- make release and visibility boundaries explicit
- build trust before interpretation deepens

### Data inputs

- invited count
- completed count
- response coverage
- minimum-n thresholds
- release state
- suppressed layer notes

### Layout composition

- response strip or response panel as primary visual
- governance boundary module as secondary anchor
- compact annotation layer

### Visual modules

- response strip
- governance boundary module
- threshold annotation block

### Copy blocks

- short explanation of safe visibility
- what may and may not yet be shown

### Governance cues

- `Wat getoond mag worden`
- `Wat verborgen blijft`
- `Waarom dat zo is`
- `Wat je hier niet uit mag concluderen`

### Forbidden elements

- segment rows
- hidden detail implied by blank space
- soft privacy boilerplate only

### Acceptance criteria

- response basis is immediately legible
- governance module feels premium and intentional
- page creates trust rather than friction

---

## 4.4 Page 4 — Loep Culture Index Hero

### Page title

`Loep Culture Index`

### Purpose

- establish the index as the report’s visual center
- make its meaning clear without overclaim

### Data inputs

- Loep Culture Index score
- short interpretation line
- brief domain context

### Layout composition

- one dominant hero visual
- short interpretation block
- one reading-boundary annotation

### Visual modules

- hero signal panel
- interpretation text block
- governance annotation

### Copy blocks

- what the index is
- what the index is not
- why it should be read with domains and governance

### Governance cues

- explicitly no benchmark
- explicitly no pass/fail
- explicitly no overall culture verdict

### Forbidden elements

- gauge
- speedometer
- traffic-light health widget
- benchmark-position graphic
- pass/fail framing

### Acceptance criteria

- Culture Index is visually premium and non-dashboard-like
- the page cannot be mistaken for a health-score screen
- reader understands index as navigation signal

---

## 4.5 Page 5 — Board Attention Points

### Page title

`Board attention points`

### Purpose

- make the report directly useful in a boardroom discussion
- surface the 3–5 most relevant bounded attention lines

### Data inputs

- board attention points
- priority reasons
- confidence labels
- what to verify next

### Layout composition

- 3–5 editorial attention cards
- strong visual hierarchy
- enough space around each card to feel deliberate

### Visual modules

- attention cards
- confidence chips
- verify-next microblocks

### Copy blocks

- short card headline
- priority reason
- what to verify next

### Governance cues

- confidence explains stability, not certainty
- no causal diagnosis

### Forbidden elements

- bullet-only list
- action checklist
- intervention platform framing
- blame language

### Acceptance criteria

- page feels like the heart of the report
- first five pages together now feel premium
- attention cards are more compelling than any surrounding data detail

---

## 4.6 Page 6 — Domain Profile

### Page title

`Domeinbeeld`

### Purpose

- show the broad organizational profile across the ten fixed domains
- support reading order without creating ranking behavior

### Data inputs

- ten domain scores
- domain labels
- short domain interpretation lines

### Layout composition

- one dominant domain profile visual
- short interpretation line per domain or grouped narrative
- restrained annotation layer

### Visual modules

- editorial domain profile module
- subdued score labels
- interpretation captions

### Copy blocks

- page intro on reading sequence
- short interpretation notes for visible patterns

### Governance cues

- order is reading sequence, not ranking
- domains are descriptive, not evaluative verdicts

### Forbidden elements

- red/amber/green status coloring
- rank-number dominance
- top/bottom framing
- scoreboard styling

### Acceptance criteria

- page reads as a profile, not a ranking table
- labels are calm and readable
- short interpretation lines support the visual rather than crowd it

---

## 4.7 Page 7 — Pattern Logic

### Page title

`Patronen in samenhang`

### Purpose

- show recurring theme pairs and broader system logic
- elevate the read beyond isolated domain scores

### Data inputs

- recurring theme pairs
- related domain context
- bounded explanatory text

### Layout composition

- relationship diagram or paired narrative modules as primary center
- supporting narrative beneath or beside

### Visual modules

- pattern connector / relationship diagram

### Copy blocks

- why it matters descriptively

### Governance cues

- descriptive only
- non-causal

### Forbidden elements

- correlation matrix look
- technical systems table
- causal arrows

### Acceptance criteria

- page feels interpretive and intelligent
- not mistaken for technical analysis output
- relationships are readable in one scan

---

## 4.8 Page 8 — Segment Contrasts, If Governed and Released

### Page title

`Segmentcontrasten`

### Purpose

- show bounded safe differences only where governed release allows
- preserve trust and containment

### Data inputs

- released segment contrasts
- segment labels
- safe contrast values
- suppressed/unreleased states

### Layout composition

- compact contrast modules
- governance state visibly embedded
- suppressed states rendered intentionally

### Visual modules

- safe segment contrast module
- governance state tags
- hidden-state placeholders

### Copy blocks

- short interpretation of visible contrasts
- short note on missing/suppressed layers

### Governance cues

- only released segments may appear
- hidden/suppressed layers must still be acknowledged
- no named manager layer

### Forbidden elements

- sorted league table
- worst/best segment labels
- winner/loser colors
- blank disappearance of hidden layers

### Acceptance criteria

- contrasts feel governed, not exploratory
- hidden states are visible as governance decisions
- page does not create ranking behavior

---

## 4.9 Page 9 — Follow-On Decision

### Page title

`Vervolgrichting na de baseline`

### Purpose

- close the report with bounded route choice
- prevent open-ended action theater

### Data inputs

- follow-on outcomes
- route decision notes
- Pulse eligibility framing

### Layout composition

- route decision matrix or route cards
- clear bounded option structure
- short explanation under each route

### Visual modules

- route decision matrix
- bounded route cards

### Copy blocks

- no immediate next route
- deeper governed work
- Pulse follow-on
- another Loep route

### Governance cues

- no route opens automatically
- Pulse is bounded and conditional
- unclear baseline interpretation may not be patched with Pulse

### Forbidden elements

- workflow software feel
- task tracker framing
- intervention prescription engine

### Acceptance criteria

- the report ends with disciplined choice
- route logic feels calm and premium
- no card implies automatic continuation

---

## 4.10 Page 10 — Method & Boundaries

### Page title

`Methodiek en begrenzingen`

### Purpose

- preserve trust
- end the document with quality, not legal residue
- make boundaries explicit

### Data inputs

- method notes
- fixed domain logic
- score and threshold reminders
- what the output does not prove

### Layout composition

- compact method narrative
- structured boundary blocks
- annotation layer rather than dense appendix table

### Visual modules

- governance boundary block
- method note block
- interpretation-limit annotations

### Copy blocks

- what this measures
- what this does not prove
- how to read the output responsibly

### Governance cues

- no causal proof
- no individual prediction
- no manager ranking
- no benchmark claim in v1

### Forbidden elements

- dense technical appendix table dominating the page
- visual downgrade into plain legal text
- contradictory language about what the score means

### Acceptance criteria

- page feels designed, not apologetic
- trust remains high at the end of the report
- method and governance are legible without overwhelming the reader

---

## 5. Global Forbidden Elements

The full report must not contain:

- dashboard screenshot aesthetics
- gauge or speedometer semantics
- benchmark-position visuals
- league-table segment ranking
- winner/loser styling
- pass/fail framing
- unreleased layer amplification
- optional output framed as standard output

---

## 6. Global Acceptance Criteria

The full board report redesign is successful only if:

- pages 1–5 feel unmistakably premium
- no page looks like a dashboard export
- the Culture Index hero is clearly non-gauge and non-pass/fail
- the domain page feels like reading sequence rather than ranking
- the governance frame feels intentional and premium
- PDF export remains fully readable and visually stable
- the artifact remains fully aligned with `culture_assessment` governance and method boundaries

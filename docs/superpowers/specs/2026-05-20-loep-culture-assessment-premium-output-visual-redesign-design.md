# Loep Culture Assessment Premium Output Visual Redesign

**Date:** 2026-05-20  
**Status:** premium-output redesign design spec  
**Primary scope:** `Loep Culture Assessment / Loep Cultuurbeeld`  
**Canonical route id:** `culture_assessment`

---

## 1. Purpose

This spec defines the **Premium Output Visual Redesign** for the `Loep Culture Assessment / Loep Cultuurbeeld` output family.

Its purpose is to close the current gap between:

- a methodologically strong, governance-safe boardroom assessment

and:

- a visually convincing premium artifact family that actually feels boardroom-grade

The immediate redesign target is not the full suite and not the full application UI.

It is the visual and editorial quality of:

- the `board report PDF`
- the `boardroom PDF deck`

with the `executive one-pager` treated as a later derivative of the same visual grammar.

This redesign exists because the current sample output is not yet visually strong enough to support the premium product claim. It needs more hierarchy, stronger compositions, clearer page rhythms, and richer but disciplined visuals.

---

## 2. Strategic Role

Inside the `culture_assessment` operating system, this redesign sits inside the broader Premium Delivery stream.

Its strategic role is to ensure that the product no longer relies only on:

- methodological discipline
- governance discipline
- executive-safe copy

but also presents itself as:

- premium
- boardroom-legible
- commercially convincing
- visually distinctive versus generic survey output

This redesign does **not** change the product identity.

It exists to make the current identity visually credible:

- premium boardroom culture and engagement baseline
- annual baseline-first route
- guided and governed
- non-ranking
- non-benchmark-first
- non-self-serve

---

## 3. Core Redesign Principle

The redesign follows one primary rule:

> The output family must feel like a boutique boardroom publication, not like a dashboard export.

That means:

- visual richness is allowed
- strong layout presence is required
- charts may be prominent
- editorial rhythm should be intentional

But:

- every visual remains subordinate to executive interpretation
- no page may drift into analytics-workbench behavior
- no artifact may read like a generic SaaS PDF

The report and deck must feel like two expressions of one system:

- same narrative spine
- same visual language
- same trust boundaries
- same governance cues

---

## 4. Chosen Creative Direction

The approved direction for this redesign is:

- **artifact family:** one combined premium report+deck family
- **style mode:** editorial / magazine-like premium
- **intensity:** visually rich, not sparse
- **tone:** boutique consultancy / annual-report premium
- **color world:** warm and human
- **priority emphasis:** strongest visual energy goes to:
  - cover
  - executive read
  - Loep Culture Index
  - board attention points

This means the family should not resemble:

- a KPI dashboard printed to PDF
- a generic management PowerPoint
- an overloaded analytics report
- a traffic-light score document

## 6. Output Family Intent

The output family must now clearly differentiate three maturity layers:

### 6.1 Board report PDF

Role:

- compact premium executive read
- downloadable board-facing report
- bounded pilot-delivery artifact

Design consequence:

- visually rich enough to stand on its own
- self-readable without facilitator dependence
- concise enough to remain scannable
- more textual than the deck
- print-worthy and download-worthy as a standalone artifact
- must feel like a finished board artifact, not a draft layout

### 6.2 Boardroom PDF deck

Role:

- guided board-read companion
- presentation-ready artifact for facilitated executive use

Design consequence:

- same design language as report
- guided and facilitator-friendly
- more spacious than the report
- larger-statement oriented
- less text per slide than the report
- paced for a facilitated board-read
- not dependent on future `.pptx` automation to feel premium

### 6.3 Executive one-pager

Role:

- later derivative from the same visual grammar

Design consequence:

- the redesign must already define the visual ingredients it would use later
- but it is not a primary redesign artifact in this pass

## 7. Canonical Page Types

The family should be built from a small, repeatable set of page types.

### 7.1 Cover page

Purpose:

- establish premium tone immediately
- signal seriousness and editorial control

Must include:

- strong title treatment
- quiet but confident context line
- enough atmosphere to feel premium without turning decorative

### 7.2 Executive opener

Purpose:

- orient the reader in one glance
- state what this baseline is and what it is not

Must include:

- executive summary block
- management question
- response context
- visible governance frame

### 7.3 Signal hero page

Purpose:

- turn the `Loep Culture Index` into a visual anchor

Must include:

- one dominant hero signal treatment
- short explanatory read
- explicit interpretation boundary

Must not use:

- dashboard gauge
- speedometer
- generic KPI card layout

### 7.4 Board attention page

Purpose:

- make the artifact directly actionable in a boardroom context

Must include:

- 3-5 attention cards
- priority reason
- confidence label
- what to verify next

### 7.5 Domain profile page

Purpose:

- show the broad organization picture without ranking behavior

Must include:

- ordered domain view
- editorial visual hierarchy
- restrained comparative emphasis

### 7.6 Pattern page

Purpose:

- show recurring theme pairs and broad pattern logic

Must include:

- relationship logic
- paired-domain reading
- non-causal explanatory framing

### 7.7 Segment contrast page

Purpose:

- show governed segment differences where safe and released

Must include:

- bounded comparison view
- visible suppressie/governance logic
- no ranking tables

### 7.8 Governance boundary page or block

Purpose:

- make governance feel like a quality system, not a disclaimer dump

Must include:

- minimum-n
- hidden layers
- release state
- named manager lock
- text boundary where relevant

### 7.9 Follow-on decision page

Purpose:

- end the artifact with bounded route choice instead of open-ended action theater

Must include:

- no immediate next route
- deeper governed work
- Pulse follow-on
- another Loep route

### 7.10 Method / appendix page

Purpose:

- preserve trust without overwhelming the executive read

Must include:

- compact method notes
- what this does not prove
- governance as quality boundary

### 7.11 Board report PDF v1 sequence

The board report PDF v1 sequence is fixed as:

1. Cover
2. Executive opener
3. Response basis & governance frame
4. Loep Culture Index hero
5. Board attention points
6. Domain profile
7. Pattern logic
8. Segment contrasts, if governed and released
9. Follow-on decision
10. Method & boundaries

### 7.12 Boardroom deck v1 sequence

The boardroom deck v1 sequence is fixed as:

1. Opening / purpose
2. What was measured
3. Response and governance
4. Executive read
5. Culture Index hero
6. Domain picture
7. Board attention point 1
8. Board attention point 2
9. Board attention point 3
10. Segment contrast, if released
11. What not to conclude
12. Decision questions
13. Follow-on route options

## 9. Visual System Mini-Layer

This redesign must define a concrete visual system layer so that report and deck use the same tokens even when their pacing differs.

### 9.1 Report page format

- portrait A4
- PDF export first
- print-safe contrast and annotation density

### 9.2 Deck page format

- 16:9 landscape
- optimized for boardroom screen share and exported PDF deck use
- fewer modules per page than the report

### 9.3 Grid system

- report: 12-column editorial grid
- deck: 12-column presentation grid with wider visual spans
- all modules must snap to grid logic; no dashboard widget placement

### 9.4 Margin system

- report: generous outer margin and stable inner gutter
- deck: wider side margins and stronger top/bottom breathing room
- no dense edge-to-edge chart framing

### 9.5 Typography scale

- display tier for cover, section openers and hero signal pages
- heading tier for page titles and attention titles
- subheading tier for governance labels, module headers and confidence framing
- body tier for narrative explanation
- caption tier for annotations, source notes and interpretation boundaries

### 9.6 Color roles

- background neutral
- ink neutral
- warm accent
- secondary warm accent
- governance neutral
- subdued emphasis tint

### 9.7 Chart palette roles

- primary signal tone
- secondary comparison tone
- suppressed / hidden governance tone
- neutral context tone
- annotation tone

### 9.8 Module spacing

- large spacing between primary page modules
- medium spacing between related narrative/chart blocks
- tight spacing only inside cards or annotations
- no compressed multi-widget density

### 9.9 Annotation style

- short
- quiet
- precise
- editorial rather than technical-debug

### 9.10 Icon / line style

- simple
- restrained
- non-playful
- thin-to-medium line weights
- no decorative icon library overload

## 10. Core Visualization Rules

### 10.1 Loep Culture Index

The index must be a hero visual, not a standard score widget.

Required qualities:

- large visual anchor
- clearly premium
- descriptive, not diagnostic
- paired with explicit read boundaries

Allowed forms:

- editorial index band with large numeric anchor and interpretation text
- vertical signal column with domain context
- abstract signal field without gauge semantics

Forbidden:

- gauges
- speedometers
- traffic-light health widgets
- benchmark-position visuals
- pass/fail visuals

### 10.2 Domain picture

The domain visual must:

- support a bestuurlijke leesvolgorde
- surface the broadest signal pattern
- avoid a raw ranking-table feel

Preferred forms:

- editorial profile bars
- stepped profile compositions
- visually rich ordered bars

Domain profile rules:

- no red/amber/green status coloring
- no rank-number dominance
- domains may be ordered only as reading sequence
- show scores with subdued labels
- include a short domain interpretation line
- top/bottom framing is forbidden

### 10.3 Board attention points

These may not appear as a plain bullet list.

They must render as:

- strong narrative cards or blocks
- visually central on the page
- clearly more important than surrounding detail

### 10.4 Pattern logic

Recurring theme pairs must be shown through:

- relationship diagrams
- paired narrative blocks
- subtle connective systems

Not through:

- raw cross-tables
- technical analytics tables

### 10.5 Segment contrasts

Segment visuals must:

- remain bounded
- feel governed
- never imply a free slicing environment

Preferred forms:

- paired cards
- compact contrast columns
- small-number comparisons

Segment contrast rules:

- no sorted league table
- no worst/best segment labels
- no color-coded winners/losers
- only released segments may appear
- hidden or suppressed layers must show as governance state, not as blank absence

### 10.6 Governance boundaries

Governance content must:

- look designed
- feel authoritative
- read as quality control

It must not feel like:

- legal leftovers
- footnote clutter
- apologetic copy

The governance boundary module must include:

- `Wat getoond mag worden`
- `Wat verborgen blijft`
- `Waarom dat zo is`
- `Wat je hier niet uit mag concluderen`

## 11. Visual Style Rules

### 11.1 Typography

The family should use:

- editorial, premium hierarchy
- stronger display tone for titles and big statements
- restrained, highly readable body typography

The typography must support:

- boardroom seriousness
- human warmth
- generous visual pacing

### 11.2 Color

The color world should be:

- warm
- human
- premium
- calm

The report must avoid default SaaS traffic-light logic as its dominant emotional palette.

Accent colors should help:

- hierarchy
- emphasis
- signal differentiation

not simulate a health dashboard.

### 11.3 Spacing

The family must use:

- large breathing space
- strong primary focus per page
- low visual clutter

Every page should feel composed, not densely packed.

### 11.4 Chart behavior

All charts must feel like one family.

They must share:

- line weight behavior
- label rhythm
- annotation style
- restrained palette
- strong white-space discipline

### 11.5 Narrative behavior

Every page must answer a boardroom question.

The artifact must not rely on the reader to infer why a page exists.

Each page should have:

- a visual center
- a clear reading line
- bounded interpretation

---

## 12. Visual Redesign Governance Guardrails

Visual redesign may not:

- introduce new result fields
- change score meaning
- change threshold behavior
- visually amplify suppressed or unreleased layers
- make optional outputs appear standard

---

## 13. Anti-Patterns

The redesign must explicitly avoid:

- dashboard-export aesthetics
- widget grids
- heavy use of gauges
- generic default charts
- traffic-light overload
- ranking tables as page anchors
- dense spreadsheet pages
- consultant-template PowerPoint styling
- “all metrics on one page” layouts

If a page feels like it came straight out of a product dashboard, it fails the design intent.

---

## 14. Artifact-Specific Redesign Priorities

### 14.1 Board report PDF

This is the first and main redesign target.

Priority:

- make the first 4-5 pages undeniably premium
- strengthen cover, opener, index and board attention
- give the report stronger page rhythm and visual drama

### 14.2 Boardroom deck

This is the second target.

Priority:

- use the same visual family
- lean more presentational than the report
- preserve the same signal, domain, attention and governance logic

### 14.3 Executive one-pager

This remains a derivative target for a later pass.

This spec must support it, but not fully design it now.

## 15. Acceptance Criteria

This redesign spec is only successful if it produces a family that:

- visibly feels more premium than the current sample output
- does not read as dashboard export
- keeps `culture_assessment` method and governance truths fully intact
- creates one coherent report+deck design language
- makes the board report visually convincing enough for guided buyer and pilot contexts
- preserves bounded interpretation and no-ranking behavior

The board report is accepted only if:

- the first 5 pages feel visually premium
- no page looks like a dashboard screenshot
- the Culture Index is not gauge-like
- the domain page does not feel like ranking
- the governance block feels designed
- deck and report use the same visual tokens
- output remains readable in PDF export

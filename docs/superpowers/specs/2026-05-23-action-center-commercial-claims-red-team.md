# Action Center Commercial Claims Red-Team

## Status
Proposed closeout review aid

## Purpose

Create a bounded red-team and closeout review layer for Action Center commercial language across the 2026-05-23 standalone-readiness artifact set.

This artifact exists to help reviewers challenge overclaiming before buyer-facing reuse, internal packaging drift, or premature standalone framing appears in materials.

It is a review aid only. It does not authorize standalone commercialization, launch, route expansion, standalone pricing, separate GTM, or broader claim expansion.

## Current Product Boundary

All review under this artifact assumes the current approved product truth remains unchanged:

- Action Center remains embedded inside Loep today
- approved route families remain `exit` and `retention`
- HR remains the governance owner
- Action Center remains route-bound, post-scan only, and bounded away from generic workflow broadening
- Loep remains canonical for route truth
- Action Center remains authoritative for follow-through status, review, continuation, and closeout within approved scope

## Allowed Claims

The following claims are allowed when they are used in a way that stays explicitly inside the current product boundary:

- makes follow-through visible
- records ownership and review rhythm
- shows where follow-through stalls
- supports HR-governed action and closeout
- turns structured people-route outcomes into reviewable follow-through

## Prohibited Claims

The following claims are prohibited unless they appear only in a negative boundary, red-team, or explicit gate-block context:

- improves retention
- reduces exit
- proves intervention impact
- predicts individual risk
- measures manager effectiveness
- solves engagement
- replaces project management
- replaces HR operating systems

## Review Questions

Every closeout or red-team pass should answer these questions explicitly:

1. Does the artifact keep Action Center embedded in Loep rather than drifting into standalone framing?
2. Does the artifact stay limited to approved route families only: `exit` and `retention`?
3. Does the artifact describe governed follow-through rather than generic workflow, task, case, or HR operating-system breadth?
4. Do all value statements stay inside visibility, ownership, review rhythm, stall detection, and HR-governed closeout?
5. Are any future-state references clearly framed as internal review context rather than external authorization?

If any question cannot be answered "yes" without explanation gymnastics, the artifact should be tightened before reuse.

## Closeout Review Workflow

1. Run the prohibited-language scan across the 2026-05-23 Action Center spec set.
2. Confirm every match appears only in a prohibited-claim list, negative boundary, or explicit gate block.
3. Run the placeholder scan for the exact placeholder terms `TBD`, `TODO`, `implement later`, `fill in`, and `appropriate` across the 2026-05-23 artifact set and the implementation plan.
4. Confirm no open placeholder language remains in the artifacts themselves.
5. Re-read every covered artifact in this batch together to confirm none of them authorizes launch, route expansion, standalone pricing, separate GTM, or broader claim drift.
6. Record the verification notes in the implementation plan closeout section.

## Closeout Record Template

Use the following closeout structure each time this review is run:

| Check | Scope | Result | Exceptions |
| --- | --- | --- | --- |
| prohibited-language scan | exact 2026-05-23 artifact file set reviewed | `pass` / `fail` | any allowed negative-boundary matches |
| placeholder scan | exact placeholder terms plus exact artifact file set and implementation plan | `pass` / `fail` | any instructional or self-review-only matches |
| full artifact re-read | every listed artifact in this review aid | `pass` / `fail` | any doc needing tightening before reuse |

# Loep Culture Assessment Follow-On Profile Model

**Date:** 2026-05-20  
**Status:** follow-on profile artifact  
**Primary scope:** `Loep Culture Assessment / Loep Cultuurbeeld`  
**Canonical route id:** `culture_assessment`

---

## 1. Purpose

This artifact explains how the same follow-on logic works across enterprise and MKB profiles without creating separate product logic.

---

## 2. Shared Product Logic

Across both profiles:

- no next route opens by default
- deeper governed work remains distinct from route switching
- `Pulse` remains bounded follow-on only
- `RetentieScan` and `ExitScan` remain question-led handoffs

---

## 3. Enterprise Profile

Typical enterprise behavior may include:

- more governed analysis before route switching
- more explicit approval layers
- more complex business-unit context
- more deliberate use of `Pulse` where a bounded theme and owner exist

Enterprise does not introduce:

- separate route identity
- unrestricted analyst exploration
- automatic multi-route expansion

---

## 4. MKB Profile

Typical MKB behavior may include:

- simpler route branching
- faster no-route or deeper-governed-work decisions
- narrower use of segment and follow-on logic

MKB does not introduce:

- lite product logic
- weaker governance standards
- simplified threshold invention

---

## 5. Profile Comparison

| Dimension | Enterprise | MKB |
| --- | --- | --- |
| Route branching | more explicit | simpler |
| Approval depth | more layered | fewer approvers |
| Governed work before switching | more common | narrower |
| `Pulse` follow-on use | more structured | more selective |
| Product logic | same | same |

---

## 6. Acceptance Read

This profile model is acceptable only when:

- one product logic remains intact
- profile differences stay limited to depth and branching complexity
- no lite logic is introduced

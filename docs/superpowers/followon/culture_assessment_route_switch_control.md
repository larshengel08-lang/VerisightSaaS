# Loep Culture Assessment Route Switch Control

**Date:** 2026-05-20  
**Status:** follow-on route-switch artifact  
**Primary scope:** `Loep Culture Assessment / Loep Cultuurbeeld`  
**Canonical route id:** `culture_assessment`

---

## 1. Purpose

This control defines when handoff from `culture_assessment` to another Loep route is justified.

Route switching must remain question-led, not SKU-led.

---

## 2. Allowed Switch Questions

### 2.1 RetentieScan handoff

Use only when the next useful question is primarily:

- active-population retention pressure
- stay or leave pressure in the current workforce
- a narrower retention-focused read than the broad annual baseline

### 2.2 ExitScan handoff

Use only when the next useful question is primarily:

- departure patterns
- retrospective leaving context
- structured learning from exits rather than broad organization climate

---

## 3. Not Enough To Switch

The following do not justify route switching on their own:

- a generic desire for more measurement
- one low domain score
- curiosity about segment differences
- unresolved baseline interpretation
- sales convenience

---

## 4. Route-Switch Approval

Any route switch requires explicit approval by:

- route owner
- governance reviewer
- suite reviewer

Approval must record:

- why the baseline question has shifted
- why deeper governed work is not enough
- why the selected route now fits better

---

## 5. Acceptance Read

This control is acceptable only when:

- `RetentieScan` and `ExitScan` triggers are explicit
- route-switch logic remains question-led
- default next-route behavior is absent

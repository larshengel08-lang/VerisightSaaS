# CULTUURBEELD_OUTPUT_UPLIFT_EXECUTION_PLAN

## 1. Objective

Voer een bounded premium-output uplift uit voor `Loep Culture Assessment / Loep Cultuurbeeld` binnen de bestaande productidentiteit en route `culture_assessment`.

Het doel van deze slice is:

- een overtuigende **Sample Output v2** bouwen voor fictieve organisatie `Noordhaven Industrie Groep`
- de **board report PDF** visueel en redactioneel omhoog trekken
- een sterkere **boardroom deck** sample opleveren
- een governed **HR appendix outline** vastzetten
- een cascade-safe **manager cascade handout outline** vastzetten

Deze slice moet bewijzen dat Loep Cultuurbeeld in outputkwaliteit duidelijk boven een standaard MTO-rapport kan uitsteken, zonder benchmarkclaims, platformuitbreiding of routeverwarring.

## 2. Work packages

### WP1 — Source alignment and bounded sample contract

Doel:
- de sample-output pass expliciet koppelen aan het bestaande uplift brief, de canonical route-waarheid en het gap register

Werk:
- bevestigen van de sample scenario-regels voor `Noordhaven Industrie Groep`
- bevestigen van bounded output-claims per artifact
- vastzetten welke copy en governance statements standaard versus governed zijn

### WP2 — Board Report PDF uplift

Doel:
- de bestaande report/output-pipeline benutten of gecontroleerd herbouwen waar nodig om een premiumer board report sample te maken

Werk:
- cover / executive opener / response basis / Culture Index / board attention / domain picture / patterns / governed segment page / boundaries / follow-on richting
- page flow, visual hierarchy en copy aanscherpen
- cultuurspecifieke boundedness borgen

### WP3 — Boardroom deck uplift

Doel:
- een visueel sterker, begeleid board-read artifact maken dat zuster is van het report maar duidelijk presentationaler voelt

Werk:
- 13-slide v1 sequence uitwerken
- sterke visual centers per slide
- presenter flow zonder live dashboard afhankelijkheid

### WP4 — HR appendix outline

Doel:
- governed verdieping structureren zonder verborgen lagen te lekken

Werk:
- outline vastzetten
- als haalbaar ten minste één ontworpen sample page opleveren
- expliciet aangeven wat governed blijft en wat niet zichtbaar mag zijn

### WP5 — Manager cascade handout outline

Doel:
- een communicatie- en cascade-safe artifact neerzetten, niet een analyse- of verantwoordingsdocument

Werk:
- outline vastzetten
- als haalbaar ten minste één ontworpen sample page opleveren
- expliciete anti-ranking / anti-defensiveness copyrails bewaken

### WP6 — Showcase and sample family alignment

Doel:
- zorgen dat sample artifacts, referentiedocs en showcase-registry dezelfde waarheid vertellen

Werk:
- sample family references gelijk trekken
- buyer-facing “what you receive” laag controleren
- geen mismatch tussen artifact status, docs en registry

### WP7 — Review and bounded verification

Doel:
- bewijzen dat de output uplift voldoet aan de bounded premium-lat

Werk:
- synthetische artifact checks
- copy/grensreview
- visual review pass
- korte review note met eventuele restgaten

## 6. Acceptance criteria

De slice is pas klaar wanneer aan deze criteria is voldaan:

### Board Report PDF

- alle hoofdonderdelen van de v1 board sequence zijn aanwezig
- eerste 5 pagina’s voelen premium en boardroom-readable
- geen pagina oogt als dashboard-export
- Culture Index is niet gauge-like
- domeinpagina gebruikt bestuurlijke leesvolgorde
- segmentpagina respecteert release en suppressie
- geen benchmark-, ranking-, causal-, prediction- of manager-score wording
- artifact is bruikbaar in sales zonder extra uitleg

### Boardroom Deck

- alle 13 kernslides uit de v1-sequence bestaan
- elke slide heeft één visueel centrum
- speaker flow werkt zonder live dashboard
- attention slides zijn afzonderlijk bruikbaar
- follow-on slide houdt Pulse conditioneel
- deck voelt visueler en presentationaler dan het report

### HR Appendix

- governed status is duidelijk
- deeper content lekt geen hidden segments
- outline staat vast
- als haalbaar bestaat ten minste één governed-detail sample page

### Manager Cascade Handout

- artifact ondersteunt communicatie, niet analyse
- bevat geen team- of manager-ranking
- vraagt managers niet om scores te verdedigen
- outline staat vast
- als haalbaar bestaat ten minste één cascade-safe sample page

### Family alignment

- alle artifacts gebruiken fictieve data only
- sample family spreekt dezelfde bounded waarheid
- reference docs en showcase registry spreken dezelfde outputtaal

## 7. Verification steps

Na uitvoering moet minimaal dit worden gecontroleerd:

1. artifact existence check
- sample board report bestaat
- sample deck bestaat
- appendix/cascade outlines bestaan

2. bounded content check
- geen benchmarktaal
- geen rankingtaal
- geen causale claims
- geen prediction claims
- geen manager-score taal

3. sample scenario check
- `Noordhaven Industrie Groep` is fictief en consistent
- 10 domeinen aanwezig
- 3 attention points aanwezig
- 2 veilige segmenten zichtbaar
- 1 hidden segment als governance state zichtbaar

4. family consistency check
- board report, deck, appendix, cascade en “what you receive” zijn niet onderling tegenstrijdig

5. showcase check
- sample-showcase assets verwijzen naar de juiste artifacts

6. review note
- korte note met: wat is beter, wat blijft nog visueel middelmatig, wat is expliciet buiten scope gebleven

## 8. Visual review requirements

De visual review voor deze slice moet expliciet toetsen:

- voelt het report als premium board publication in plaats van als export?
- voelt het deck als begeleid boardroom artifact in plaats van als document met slides?
- is de typografische hiërarchie sterk genoeg?
- zijn de visuals warm, beheerst en editorial in plaats van corporate dashboardachtig?
- is er genoeg scanwaarde per pagina / slide?
- zijn attention points en domain picture direct begrijpelijk?
- blijft governance zichtbaar als kwaliteitsgrens in plaats van disclaimerdump?
- voelt de output duidelijk beter dan een standaard MTO-rapport?

Visuele review mag streng zijn. “Functioneel goed” is niet genoeg; het artifact moet ook commercieel overtuigen.

## 9. Out-of-scope boundaries

Deze slice doet expliciet niet:

- geen runtime features
- geen benchmark logic
- geen app redesign
- geen manager ranking
- geen causal driver claims
- geen nieuwe measurement logic
- geen self-serve reporting
- geen bredere platform scope
- geen interactieve drilldown als nieuwe productlaag
- geen open-text browser
- geen pricing system of packaging system

Als tijdens uitvoering iets buiten deze grenzen nodig lijkt, stoppen en terugkoppelen in plaats van de scope stil te verbreden.

# Loep Culture Assessment Action & Follow-through System Implementation Plan

**Date:** 2026-05-20  
**Status:** second-wave implementation plan  
**Primary scope:** `Loep Culture Assessment / Loep Cultuurbeeld`  
**Canonical route id:** `culture_assessment`

---

## 1. Purpose

This plan translates the `Action & Follow-through System` spec into a bounded execution sequence.

Its purpose is to make the route's post-baseline follow-through:

- decision-structured
- HR-usable
- cascade-bounded
- review-rhythmed
- follow-on explicit

without turning the route into a workflow or intervention platform.

---

## 2. Execution Rules

1. follow-through must begin with decision structure, not action theater
2. HR follow-up may deepen interpretation, but not override governance
3. manager cascade remains communication support, not performance control
4. review rhythm must be explicit before Pulse eligibility is discussed
5. Pulse remains follow-on only, never default baseline continuation

---

## 3. Work Package Sequence

### WP1 - Board Decision Logic Lock

Goal:

- make the post-board-read decision structure explicit

Deliverables:

- board decision question set
- what-not-to-conclude note
- sponsorship boundary note

Dependencies:

- approved action/follow-through spec
- premium delivery implementation plan

Acceptance:

- board decision questions are explicit
- non-causal guardrails remain visible
- sponsorship boundaries are clear

### WP2 - HR Follow-up Structure Lock

Goal:

- define how HR should structure next steps after the executive read

Deliverables:

- HR follow-up note
- governed follow-up priority template
- escalation boundary note

Dependencies:

- WP1
- governed drilldown implementation plan

Acceptance:

- HR follow-up remains structured and bounded
- deeper follow-up areas remain governance-safe
- escalation boundaries are explicit

### WP3 - Manager Cascade Follow-through Lock

Goal:

- define what safe local follow-through can look like

Deliverables:

- manager cascade follow-through note
- communication prompt set
- prohibited local inference note

Dependencies:

- WP2
- text/open-comment implementation plan

Acceptance:

- cascade artifacts remain non-ranking
- prohibited local inference is explicit
- communication prompts do not create score defense behavior

### WP4 - 30/60/90 Review Rhythm Lock

Goal:

- make the post-baseline review sequence explicit

Deliverables:

- 30-day review guide
- 60-day review guide
- 90-day review guide

Dependencies:

- WP2

Acceptance:

- each review point has a clear purpose
- no review point implies unsupported causal certainty
- governance drift checks are explicit

### WP5 - Follow-on Route Decision Control

Goal:

- define when the route stays put, deepens, or hands off to Pulse

Deliverables:

- follow-on route decision guide
- Pulse eligibility note
- non-Pulse alternative note

Dependencies:

- WP4
- listening operations implementation plan

Acceptance:

- Pulse eligibility remains bounded and explicit
- no implied default Pulse continuation exists
- alternatives to Pulse are explicit

### WP6 - Follow-through Readiness Review

Goal:

- confirm the route can support disciplined follow-through without becoming workflow software

Deliverables:

- follow-through readiness review
- unresolved risk note
- next-wave unlock note

Dependencies:

- WP3
- WP4
- WP5

Acceptance:

- decision logic, HR structure, cascade boundaries, and follow-on rules are coherent
- unresolved risks are explicit
- the stream clearly supports later follow-on architecture maturity

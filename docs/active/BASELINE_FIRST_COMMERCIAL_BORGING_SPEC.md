# BASELINE_FIRST_COMMERCIAL_BORGING_SPEC

Last updated: 2026-05-06  
Status: active  
Primary source of truth: COMMERCIAL_DEFAULT_MODEL.md  
Supporting references: product family model, packaging and route logic, pricing and packaging program plan, current website and sales surfaces.

## 1. Summary

Deze spec borgt het baseline-first commerciele model over:

- sales
- pricing
- website copy
- proposalstructuur
- interne houvast voor founder-led verkoop en productkeuzes

Doel:

- de eerste koop lichter en scherper maken
- Action Center publiek verkoopbaar houden zonder de koop te verzwaren
- live, ritme en bounded vervolgroutes ondergeschikt houden
- met zo min mogelijk onnodige verbouwing van de huidige website en saleslaag

## 2. Core commercial decision

De leidende commerciele standaard is:

- `Baseline` is de default eerste koop
- `Action Center Start` is een publieke optionele add-on
- `Live Start` en `Reviewcadans` zijn latere vervolgvormen

Belangrijke buyer-facing volgorde:

1. `Wat speelt er en wat vraagt eerst aandacht?`
2. `Hoe borgen we de gekozen opvolging zichtbaar en toegewezen?`
3. `Hoe blijven we herijken of die opvolging werkt?`

## 3. Required sales behavior

### Discovery

Sales mag niet starten vanuit vormkeuze.

Dus niet:

- `Wilt u scan, live, dashboard, action center of review?`

Wel:

- eerst de managementvraag openen
- daarna de route kiezen
- daarna pas de vorm vaststellen

### Mandatory talk track

De vaste gesprekslijn wordt:

- `Op basis van uw vraag is dit de meest logische eerste route.`
- `Daarna kunnen we de gekozen opvolging zichtbaar borgen.`
- `En pas daarna voegen we ritme toe als dat echt nodig is.`

### Proposal sequence

Elke proposal of offerte gebruikt exact deze volgorde:

1. `Baseline`
2. `Action Center Start` optioneel
3. `Vervolg / Reviewcadans` optioneel of op aanvraag

## 4. Pricing changes to keep or introduce

### Public main pricing

Publiek zichtbaar houden of brengen naar:

- `ExitScan Baseline - vanaf EUR 4.500`
- `RetentieScan Baseline - vanaf EUR 4.500`
- `Onboarding 30-60-90 Baseline - vanaf EUR 4.500`

### Public included list

Op pricing cards compact houden:

- intake
- scan
- dashboard
- managementrapport

Belangrijke regel:

- `review` niet als zwaar vijfde pricing-line-item op de hoofdkaart zetten
- wel expliciet meenemen in ondersteunende copy, productpagina en proposal

### Public add-on

Publiek zichtbaar maken:

- `Action Center Start - vanaf EUR 1.250`

Met compacte uitleg:

- een gekozen opvolgscope
- een of enkele managers of eigenaren
- beperkte actieopvolging
- status zichtbaar maken
- een reviewmoment

Belangrijke regel:

- Action Center Start blijft begrensd tot een gekozen opvolgscope
- Action Center Start is geen brede workflowlaag
- Action Center Start is geen taakmanagementlaag

### Public follow-on

Lager op de pagina, onder `Vervolg op aanvraag`:

- `ExitScan Live Start`
- `Reviewcadans`
- eventueel bounded vervolgroutes op aanvraag

Belangrijke regel:

- geen live/ritme hoofdkaart boven de fold
- geen Action Center-hoofdkaart naast de baselinekaarten

## 5. Website copy changes

## Homepage

Doel:

- ExitScan en RetentieScan blijven de twee hoofdinstappen
- Action Center blijft ondersteunend bewijs

Wijzigen:

- Action Center alleen noemen als bewijs dat Verisight niet stopt bij inzicht
- niet framen als volledig workflowplatform
- niet framen als standaard onderdeel van de eerste koop
- expliciet framen als opvolglaag voor een gekozen vervolgrichting, niet als breed workflowplatform

## Producten

Doel:

- Baseline centraal
- Action Center Start zichtbaar maar secundair

Wijzigen:

- ExitScan Baseline en RetentieScan Baseline als primaire keuzehulp
- Onboarding 30-60-90 Baseline secundair zichtbaar houden
- Action Center Start toevoegen onder `Optionele uitbreiding`
- Pulse, Leadership, Live Start en Reviewcadans onder vervolgroutes laten vallen

## Tarieven

Doel:

- pricing begrijpelijker maken zonder volledige herbouw

Vaste sectievolgorde:

1. `Eerste koop`
2. `Optionele uitbreiding`
3. `Vervolg op aanvraag`

Wijzigen:

- baselineprijzen bovenaan
- Action Center Start in een aparte uitbreidingssectie
- live, ritme, pulse en leadership lager en rustiger
- Segment Deep Dive niet terugbrengen

## Aanpak

Doel:

- de routeflow buyer-facing gelijk maken aan het baseline-first model

Nieuwe verhaalvolgorde:

1. routekeuze
2. baseline
3. dashboard + managementrapport
4. review + eerste vervolgrichting
5. optioneel Action Center Start
6. later reviewcadans of ritme

Wijzigen:

- geen vaste suggestie dat Action Center altijd al onderdeel van de eerste koop is
- geen zware assisted trajecttaal als default

## 6. Sales asset changes

### Sales Product Decision Tree

Moet expliciet vastleggen:

- baseline is de default eerste koop
- Action Center Start is optionele add-on
- Live Start en Reviewcadans zijn later vervolg
- Onboarding is een secondary first-buy, geen derde hoofdwedge

### Proposal spines

Moeten buyer-facing line items normaliseren naar:

- `ExitScan Baseline`
- `Action Center Start` optioneel
- `Reviewcadans` optioneel of op aanvraag

of equivalent per route.

### One-pagers

Moeten allemaal dezelfde logica vertellen:

- eerst de managementvraag
- dan baseline
- dan optionele opvolging
- dan ritme

### Founder-led sales support

Er moet een korte interne houvast blijven voor founder-led verkoop waarin expliciet staat:

- wat verkoop je eerst
- wat verkoop je niet eerst
- hoe positioneer je Action Center
- wanneer open je Pulse of Leadership wel of niet

## 7. Internal helper document

Aanbevolen intern hulpdocument:

- `COMMERCIAL_DEFAULT_MODEL.md`

Dit document moet de vaste geheugensteun zijn voor:

- routehierarchie
- pricinghierarchie
- websitehierarchie
- salesregels

Minimale interne regels:

- Baseline is default eerste koop
- Action Center Start is add-on, geen kernproduct
- Live Start is alleen alternatief wanneer een retrospectieve baseline niet logisch is
- Reviewcadans is recurring vervolg, geen instap
- Pulse en Leadership zijn bounded vervolgroutes
- Onboarding is secondary first-buy, geen derde hoofdwedge

## 8. Product and UI guardrails

### Dashboard

Moet analytisch blijven.

Niet standaard buyer-facing of productmatig verzwaren met:

- owners
- assignments
- reviewstructuren
- Action Center-logica als default

Wel:

- wat eerst aandacht vraagt
- eerste vervolgrichting
- optionele route naar Action Center

### Action Center

Moet productmatig alleen actief worden wanneer:

- add-on geactiveerd is
- of een bewuste vervolgvorm is gestart

### Routebeheer

Moet per traject expliciet kunnen tonen:

- route
- meetvorm
- Action Center-status

## 9. Minimum change principle

Deze borging is expliciet bedoeld om met beperkte wijzigingen te landen.

Dus:

- geen brede websiteherbouw
- geen nieuwe productstructuur
- geen extra checkout- of billinglaag
- geen los Action Center-platformverhaal

Wel:

- scherpere hierarchie
- helderdere pricingvolgorde
- normalisatie van sales language
- een canonisch intern model voor consistentie

## 10. Definition of done

Deze borging is pas klaar als:

- websitecopy overal baseline-first leest
- Action Center publiek zichtbaar is als optionele uitbreiding, niet als hoofdproduct
- pricing de volgorde `eerste koop -> optionele uitbreiding -> vervolg` volgt
- `/tarieven` vanaf-prijzen toont voor Baseline en Action Center Start zonder Action Center als hoofdproduct te presenteren
- salesdocs dezelfde lineaire keuzevolgorde gebruiken
- proposals Baseline als hoofdregel bevatten en Action Center Start alleen als optionele line item tonen
- homepage Action Center alleen noemt als follow-through bewijs, niet als eerste koop
- dashboard en product UI Action Center-status alleen tonen als add-on of vervolgvorm expliciet actief is
- Onboarding publiek secondary first-buy blijft
- Pulse en Leadership geen onnodige eerste keuzelast toevoegen
